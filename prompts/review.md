# Code Review Prompt

## 规则

1. 客观评价，不人身攻击
2. 区分"必须修复"和"建议改进"
3. 提供具体修改建议
4. 引用规范文档

## Review 维度

### 正确性
- Bug 风险
- 边界条件处理
- 错误处理

### 可读性
- 命名规范
- 注释质量
- 函数长度

### 性能
- 是否有性能问题
- 是否有更好的算法

### 架构
- 是否符合 `API_BOUNDARY.md`
- 是否符合 `CODING_STYLE.md`
- 是否违反设计原则

## Severities

| 级别 | 说明 |
|------|------|
| 🔴 Must Fix | 必须修复，否则无法合并 |
| 🟡 Should Fix | 建议修复，好的实践 |
| 🟢 Nit | 小优化，可忽略 |
