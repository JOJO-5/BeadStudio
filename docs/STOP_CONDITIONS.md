# Stop Conditions

以下情况**必须停止执行**并等待用户确认：

| 情况 | 说明 |
|------|------|
| 需要新增依赖 | 添加新 npm 包必须确认 |
| 修改架构 | 改动 API_BOUNDARY.md 定义的层级关系 |
| 修改文档规范 | 改动 CODING_STYLE.md / AGENTS.md 等规范文件 |
| 修改公共 API | 改动 engine/store/worker 的接口签名 |
| 删除已有代码 | 删除文件或清空代码必须确认 |
| Task 超出范围 | 当前 Task 预估超过 2 小时工作量 |
| 无法通过测试 | 测试失败且无法快速定位原因 |
| 两种以上方案 | 存在多个实现方案且无法判断优劣 |
| 文档冲突 | 不同文档对同一问题描述矛盾 |

---

## 停止时输出

```
## ⚠️ Stopped

**触发条件**: [具体条件]

**当前状态**:
- Phase: Phase 1
- Task: Task-014
- 已完成: [列出已完成的工作]

**问题描述**:
[具体问题]

**需要确认**:
- [ ] 选项1
- [ ] 选项2
```

---

## 恢复条件

以下情况可以自动恢复执行：

- 用户明确回复确认
- 用户提供了具体修改方案
- 用户解决了文档冲突
