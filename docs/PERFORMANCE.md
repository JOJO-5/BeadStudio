# 性能预算 (Performance Budget)

## 目标

| 指标 | 目标 | 最大可接受 |
|------|------|-----------|
| 首次打开 (FCP) | < 1s | 2s |
| 首次交互 (TTI) | < 1.5s | 3s |
| Lighthouse Performance | > 90 | > 80 |

---

## 图片处理性能预算

| 图片尺寸 | 最大耗时 | 说明 |
|----------|----------|------|
| 64×64 | < 30ms | Worker 内处理 |
| 128×128 | < 80ms | Worker 内处理 |
| 256×256 | < 300ms | Worker 内处理 |
| 512×512 | < 1s | 需要进度提示 |
| 1024×1024 | < 3s | 需要进度提示 |

**超过 2048px 的图片先压缩再处理。**

---

## 导出性能预算

| 格式 | 100×100 板子 | 200×200 板子 |
|------|--------------|--------------|
| PNG | < 500ms | < 2s |
| PDF (1页) | < 1s | < 3s |
| PDF (4页) | < 3s | < 8s |

---

## 内存限制

- 单张图片最大内存：50MB
- Worker 内存限制：100MB
- 超出限制先压缩

---

## UI 响应要求

| 操作 | 最大响应时间 |
|------|--------------|
| 按钮点击 | < 16ms（1 帧） |
| 滑块拖动 | < 16ms |
| 缩放 | < 100ms |
| 平移 | < 16ms |
| 调色板切换 | < 200ms |

---

## 帧率要求

- Canvas 渲染：稳定 60fps
- 缩放/平移时：最低 30fps
- UI 动画：60fps

---

## 优化策略

### 图片处理

1. 所有 CPU 密集操作在 Worker 中执行
2. 使用 Pica 做高质量缩放
3. 图片超过 2048px 先压缩
4. 实时预览用低分辨率草稿

### 渲染优化

1. Canvas 脏区域更新，不重绘全屏
2. 缩放级别低时减少细节绘制
3. 网格线用 CSS 而非 Canvas 绘制

### 内存优化

1. 不保存中间状态，只保留必要数据
2. 及时释放 ImageData 引用
3. Worker 处理完立即传回结果

---

## 监控

如发现性能问题，先用 Chrome DevTools Performance 面板定位瓶颈。

常见瓶颈：
- UI 线程阻塞（把任务移到 Worker）
- Canvas 重绘过多（减少绘制频率）
- 内存泄漏（检查对象引用）
